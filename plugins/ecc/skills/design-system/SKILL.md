# 设计系统 — 生成与审查视觉系统

## 何时使用

- 启动一个需要设计系统的新项目
- 审查现有代码库的视觉一致性
- 重新设计之前 — 先了解当前现状
- 当 UI 看起来"不对劲"但无法确定原因时
- 审查涉及样式变更的 PR

## 工作原理

### 模式 1：生成设计系统

分析你的代码库并生成一致的设计系统：

```
1. 扫描 CSS/Tailwind/styled-components 中的现有模式
2. 提取：颜色、字体排版、间距、圆角、阴影、断点
3. 研究 3 个竞品网站获取灵感（通过浏览器 MCP）
4. 提出设计令牌集（JSON + CSS 自定义属性）
5. 生成 DESIGN.md 并附上每个决策的理由
6. 创建交互式 HTML 预览页面（自包含，无依赖）
```

输出：`DESIGN.md` + `design-tokens.json` + `design-preview.html`

### 模式 2：视觉审查

从 10 个维度对你的 UI 进行评分（每项 0-10 分）：

```
1. 颜色一致性 — 你是使用调色板还是随机的十六进制值？
2. 字体排版层次 — 清晰的 h1 > h2 > h3 > 正文 > 说明文字？
3. 间距节奏 — 统一的缩放比例（4px/8px/16px）还是随意的？
4. 组件一致性 — 相似的元素是否外观一致？
5. 响应式行为 — 在不同断点下是流畅的还是断裂的？
6. 深色模式 — 完整实现还是半成品？
7. 动画效果 — 有目的的还是多余的？
8. 无障碍性 — 对比度、焦点状态、触摸目标
9. 信息密度 — 是拥挤还是清爽？
10. 精致度 — 悬停状态、过渡效果、加载状态、空状态
```

每个维度都会给出评分、具体示例以及带有精确 file:line 的修复建议。

### 模式 3：AI 模板化检测

识别千篇一律的 AI 生成设计模式：

```
- 到处使用无意义的渐变
- 紫色到蓝色的默认配色
- 毫无意义的"玻璃拟态"卡片
- 不该有圆角的地方使用圆角
- 滚动时过度的动画效果
- 在通用渐变背景上居中文字的标准横幅
- 毫无个性的无衬线字体堆叠
```

## 示例

**为 SaaS 应用生成设计系统：**
```
/design-system generate --style minimal --palette earth-tones
```

**审查现有 UI：**
```
/design-system audit --url http://localhost:3000 --pages / /pricing /docs
```

**检查 AI 模板化设计：**
```
/design-system slop-check
```
